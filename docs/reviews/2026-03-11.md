# Oversteer Code Review (2026-03-11)

Scope: review of the current single-file game implementation in `arena-drifter/index.html` plus obvious project/asset issues that affect gameplay and shipping quality.

## What's Working Well

- Clear sectioning and "module-ish" organization inside a single file (CFG / Input / Physics / Player / Enemy / Props / FX / Camera / Trail / Waves / Upgrades / HUD / Game).
- Performance-minded patterns already in place in many systems (swap-and-pop removals, ring buffers, pre-rendered FX via `FXCache`, `PerfMon` instrumentation).
- Core loop is coherent: drift -> near-miss/combos -> trail loops -> wave breaks -> upgrades -> difficulty ramps.

## Blockers / P0 Issues (Fix Before Content Work)

- Mobile can't reliably start a run from the menu.
  - `Game.update()` only starts on keyboard (`Input.enter` or `Input.keys['Space']`) while touch input never sets either (`arena-drifter/index.html` around `Game.update()` ~2727).
  - Impact: "mobile support" is effectively non-functional before wave 1.
  - Fix: treat any touch in MENU as "start" (or add a dedicated "tap to start" hit region/button).

- Mobile can't select upgrades (run blocks after wave 1).
  - Upgrade selection is keyboard-only (`Digit1/2/3`) and there's no touch hit-testing for cards/reroll (`arena-drifter/index.html` in `STATE.UPGRADE` handling ~2806 and `renderUpgradeBreak()` ~2539).
  - Impact: hard blocker for mobile play.
  - Fix: store card bounds during `renderUpgradeBreak()` (or compute deterministically) and handle `touchend` -> select card / reroll.

- Pickup processing returns early (multi-pickup-per-frame bug + boost zones skipped).
  - In `Waves.update()`, scrap collection does `return pType;` inside the scrap loop (`arena-drifter/index.html:2123`).
  - Boost-zone collection also returns early (`arena-drifter/index.html` in boost-zone loop shortly after).
  - Impact: magnet + high-speed runs silently lose pickups; boost zones may not trigger if a scrap is collected the same frame.
  - Fix: collect all events for the frame and return a list/bitmask, or continue scanning and return the "highest priority" event after loops finish.

- `Speed Demon` enemy-speed downside is effectively broken.
  - `applyUpgrade()` does `this.speedBonus += 40;` when `speed_demon` is picked (`arena-drifter/index.html` ~2717).
  - Later, "Progressive intensity" overwrites `this.speedBonus` every frame based on score (`arena-drifter/index.html` ~3145).
  - Impact: the "enemies +10% speed" part of the upgrade is inconsistent or completely negated.
  - Fix: split into two values (e.g. `speedBonusFromScore` + `speedBonusFromUpgrades`) and sum them when spawning enemies.

- Trail upgrades don't persist across waves.
  - Upgrades like `wider_trail` and `trail_echo` directly mutate `Trail.CLOSE_DIST` and `Trail.MAX_POINTS` (`arena-drifter/index.html` ~2378-2384).
  - Wave transition calls `Trail.reset()` which restores `MAX_POINTS=400` and `CLOSE_DIST=40` (`arena-drifter/index.html` ~2065; `Trail.reset()` ~1630).
  - Impact: trail-related upgrades are reset each wave (or only apply inside a single combat phase), which likely isn't intended.
  - Fix: separate "clear points" from "reset tuning" (e.g. `Trail.clear()` vs `Trail.resetAll()`), or re-apply tuning from player upgrades on wave start.

## Performance Issues (High Value)

- `shadowBlur` is still used for scraps and boost zones.
  - `Waves.renderScraps()` uses `ctx.shadowBlur` (`arena-drifter/index.html:2285`).
  - `Waves.renderBoostZones()` uses `ctx.shadowBlur` (`arena-drifter/index.html:2337`).
  - Impact: avoids the `FXCache` win you already achieved elsewhere; can tank FPS on weaker/mobile GPUs.
  - Fix: convert these to pre-rendered glow sprites/canvases in `FXCache` (same pattern as props/vignette).

- Trail rendering is O(segments) draw calls per frame (twice).
  - `Trail.render()` does `beginPath()/stroke()` per segment for glow + main pass (`arena-drifter/index.html` in `Trail.render()` ~1810+).
  - Impact: large unnecessary draw-call count when trail is long.
  - Fix: batch segments into 1-2 paths per pass (one per color, since combo color changes at thresholds).

- Props collision scans the entire prop list each frame.
  - `Props.checkPlayerCollision()` iterates `allProps` and allocates a new `hits` array (`arena-drifter/index.html` ~1150+).
  - Impact: currently OK at low density, but will scale poorly if prop density/world size increases.
  - Fix: use the existing chunk grid for collision queries (check only nearby chunks), and reuse a scratch array to reduce GC.

## UX / Consistency Issues (Medium)

- Touch drift UX mismatch: UI shows a drift button, but input treats any right-half touch as drift.
  - `Input._touchStart()` uses `p.x < CFG.W/2` for stick vs drift (`arena-drifter/index.html` ~430-460), while `Game.touchDriftBtn` suggests a specific button region (`arena-drifter/index.html` ~2657 and `renderTouchUI()` ~3715).
  - Impact: confusing expectations; accidental drifting is easier than intended.
  - Fix: if you're drawing a button, hit-test against it (or stop drawing a button and label the right side as drift zone).

- Docs/control mismatch risk.
  - Menu shows `SPACE (hold)` for drift (`arena-drifter/index.html` in `renderMenu()` ~3680).
  - `README.md` controls table doesn't mention Space drift (it mentions `S/Down` reverse/handbrake).
  - Impact: players reading the README may miss core input.
  - Fix: align README with in-game menu controls.

- Menu prompt is keyboard-only.
  - `renderMenu()` displays `[ ENTER ] TO START` but not "tap to start" even though touch is claimed to be supported.
  - Fix: show "tap to start" when touch is detected or always show both prompts.

## Asset / Content Issues

- Missing prop asset referenced in config.
  - `CFG.PROP_POOL` references `props/puddle_1.png` (`arena-drifter/index.html` near the top).
  - `arena-drifter/assets/props/` does not contain `puddle_1.png`.
  - Impact: rendering falls back to rectangles; also "slip" gameplay may feel uncommunicated visually.
  - Fix: add the PNG or remove/replace the entry.

- No audio implementation yet (by design).
  - No `Howler`/`AudioManager` references exist in `arena-drifter/index.html`.
  - Impact: your PRD correctly flags this as the single highest-impact missing feature for a v1.0 release.

## Maintainability / Testing Gaps

- Single-file approach is still fine at this size, but the most bug-prone logic is currently hard to regression-test.
  - Suggested first tests (from `docs/unit-testing/initial-testing-plan.md`): encirclement geometry, pickup collection (multi-collect), upgrade/touch hit-testing, early wave spawning rules.
  - Minimal path: add `test/` + `node --test` for pure functions (geometry + hit-testing) extracted into a tiny shared file (or duplicated for tests if you want to keep the game file pristine).

## Suggested Fix Order (Fastest Path to "Shippable v1.0")

1. Fix pickup early-return + change pickup return type to allow multiple events per frame (`arena-drifter/index.html:2123`).
2. Implement touch "tap to start" and upgrade card touch selection (MENU + UPGRADE states).
3. Fix `Speed Demon` + `speedBonus` overwrite bug (separate additive components).
4. Split `Trail.reset()` into "clear points" vs "reset tuning" so trail upgrades persist.
5. Remove remaining `shadowBlur` usages and batch trail rendering.
6. Add/replace `puddle_1.png` so slip hazards read visually.
